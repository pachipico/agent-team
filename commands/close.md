# /close — 세션 종료

> 현재 작업 상태를 정리하고 다음 세션에서 이어갈 수 있도록 STATE.md를 업데이트한다.

---

## 실행

> 📋 [Orchestrator] 세션을 정리합니다.

---

## Step 1: 미완료 작업 확인

```
1. STATE.md 읽기
2. 현재 페이즈 디렉토리에서 PLAN/SUMMARY 파일 확인
3. git status로 미커밋 변경사항 확인
```

---

## Step 2: 상태별 안내

### 미커밋 변경사항이 있을 때:
```
⚠️ 미커밋 변경사항이 있습니다:

[git status 결과 요약]

권장: 커밋 후 종료하거나, 다음 세션에서 커밋하세요.
```

### Worker 구현 완료 + 커밋 미완료:
```
📌 Phase [N] Plan [M] — 구현 완료, 커밋 대기 상태로 종료합니다.
   다음 세션: 커밋 후 /execute → Tester 검증
```

### Plan 작성 완료 + 실행 전:
```
📌 Phase [N] Plan [M] — PLAN 작성 완료, 미실행 상태로 종료합니다.
   다음 세션: /execute → Worker 구현
```

### Tester 검증 완료:
```
📌 Phase [N] Plan [M] — 검증 완료 상태로 종료합니다.
   다음 세션: /execute → 다음 plan
```

### 페이즈 전체 plan 완료:
```
📌 Phase [N] — 모든 plan 완료, 페이즈 검증 대기 상태로 종료합니다.
   다음 세션: /phase-review → 전수 검증
```

---

## Step 3: STATE.md 업데이트

STATE.md에 아래 내용을 반영한다:

```
stopped_at: [현재 정확한 상태 설명]
last_updated: [현재 날짜시간]
```

기존 섹션 제목 언어 보존. frontmatter와 본문 동기화.

**"다음 작업" 섹션을 구체적으로 갱신한다:**

```markdown
## 다음 작업

다음 세션 시작 시: /status로 현재 상태 확인 후 /execute 실행
구체적으로: [정확히 다음에 해야 할 것]
```

---

## Step 4: 종료 보고

```
═══════════════════════════════════════
📋 세션 종료
═══════════════════════════════════════

■ 현재 위치
  Phase: [N] — [이름]
  Plan:  [M] — [제목]
  상태:  [구현 완료/커밋 대기/검증 완료/...]

■ 이번 세션 작업 요약
  - [완료한 작업 1]
  - [완료한 작업 2]

■ 미커밋 변경
  [있음 (N개 파일) / 없음]

■ 다음 세션
  1. /status 로 상태 확인
  2. [구체적 다음 액션]

═══════════════════════════════════════
```

---

## 주의사항

- STATE.md 업데이트 외에 다른 파일은 수정하지 않는다.
- 미커밋 변경사항을 강제로 커밋하지 않는다. 안내만 한다.
- 다음 세션에서 `/status`를 먼저 실행하도록 안내한다.
