# /status — 전체 상황 보고

> 읽기 전용. 프로젝트 전체 진행 현황을 한눈에 보여준다.

---

## 실행

> 📋 [Orchestrator] 프로젝트 상태를 조회합니다.

읽기: `STATE.md`, `ROADMAP.md`, 각 페이즈 SUMMARY/VALIDATION 파일

---

## 출력

```
═══════════════════════════════════════
📊 [프로젝트명] — 상태
═══════════════════════════════════════

■ 진행률: [완료]/[전체] phases

■ 현재 위치
  Phase: [N] — [이름] ([상태])
  Plan:  [M] — [제목] ([상태])

■ 페이즈 현황
  [✅] Phase 1: [이름] — 완료
  [🔄] Phase 2: [이름] — 진행 중 (plan [M]/[N])
  [⏳] Phase 3: [이름] — 대기

■ 현재 페이즈 상세
  Plan 01: ✅ passed
  Plan 02: 🔧 구현 완료 (커밋 대기)
  Plan 03: ⏳ 미작성

■ 다음 액션
  → [커맨드 안내]

═══════════════════════════════════════
```

| 상태 | 안내 |
|------|------|
| plan 없음 | `/execute` → plan 작성 |
| plan 미실행 | `/execute` → 구현 |
| 구현 완료 | 커밋 후 `/execute` → 검증 |
| 페이즈 전 plan 완료 | `/phase-review` |
| 모든 페이즈 완료 | `/verify-all` |

파일 수정 없음.
